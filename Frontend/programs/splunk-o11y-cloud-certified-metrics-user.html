Program Overview

  • ▹ Splunk Observability Cloud metrics workflow training
  • ▹ OpenTelemetry Collector deployment and configuration
  • ▹ Metrics ingestion and analytics for real-time insights
  • ▹ Visualization, detectors, and alert design
  • ▹ Efficient dashboard creation for multiple environments

Who Should Attend

Observability engineers, SREs, DevOps professionals, platform teams, and application support engineers using Splunk Observability Cloud for metrics-based monitoring.

Prerequisites

Basic familiarity with cloud-native monitoring, metrics concepts, and Splunk or observability workflows is recommended. Some exposure to OpenTelemetry and dashboards is helpful but not required.

What You Get

Hands-on O11y labs, OpenTelemetry Collector configuration exercises, detector tuning practice, dashboard templates, mock exams, and real-world monitoring scenarios.

Course Curriculum

Intermediate 6-10 week training focused on Splunk O11y Cloud metrics, OpenTelemetry, analytics, detectors, and scalable dashboarding.

Module 1-2: O11y Cloud Foundations (Weeks 1-2)

Build the foundation for effective metrics-driven monitoring in Splunk Observability Cloud.

Observability Concepts

Metrics, traces, logs · Real-time vs historical · O11y Cloud architecture · Signals, dimensions, metadata

Metrics Data Model

Time-series concepts · Metric naming · Dimension strategy · Cardinality awareness · Data hygiene

Platform Navigation

O11y Cloud interface · Charts and metric explorers · Service/infra views · Teams, access, workspaces

Module 3-4: OpenTelemetry Collector (Weeks 3-4)

Deploy and configure the OpenTelemetry Collector to send metrics from every layer of the environment.

Collector Architecture

Receivers, processors, exporters · Agent vs gateway · Pipelines for metrics · Configuration structure

Deployment & Configuration

Installing in VMs/containers · Host, app, infrastructure inputs · Exporting to O11y Cloud · Troubleshooting telemetry

Operational Best Practices

Resource sizing · Filtering for cost efficiency · Secure deployment · Maintaining configs across environments

Module 5-6: Metrics Analytics & Insights (Weeks 5-6)

Use analytics to turn raw metrics into actionable operational insight.

Metric Analysis

Aggregation, rollups, stats · Filtering by dimensions · Comparative analysis across layers · Baseline/anomaly interpretation

Finding Insights

Spotting saturation, latency, error patterns · Correlating infra and app metrics · High-cardinality areas · Capacity planning

Efficient Querying

Reducing noisy queries · Chart resolution choices · Working at scale · Organizing common analysis patterns

Module 7-8: Detectors & Alerting (Weeks 7-8)

Create useful detectors that surface issues quickly without flooding teams with noise.

Detector Fundamentals

Static, anomaly, predictive types · Threshold design · Multi-signal alerting · Lifecycle and ownership

Alert Tuning

Reducing false positives · Windowing, delays, sensitivity · Environment-specific adjustments · Response mapping

Notification Workflows

Routing alerts · On-call integration · Escalation design · Testing and maintenance

Module 9-10: Dashboards & Visualization (Weeks 9-10)

Build efficient dashboards that help teams monitor health, investigate issues, and communicate clearly.

Metric Visualization

Choosing chart types · Comparing dimensions · Breakdowns and overlays · Fast operational reading

Dashboard Design

Structuring for service ownership · Reusable layouts · Low-noise metric views · Balancing detail/readability

Operational Dashboards

SRE, DevOps, engineering dashboards · Service health, latency, throughput · Executive vs technical panels · Maintenance

Module 11-12: Capstone & Exam Prep (Weeks 11-12)

Bring together OpenTelemetry, analytics, detectors, and dashboards in a complete observability workflow.

Metrics Monitoring Capstone

Deploy Collector pipelines · Ingest metrics · Create detectors and alert routing · Build efficient dashboards

Practice Exams

Full-length O11y metrics mock tests · Scenario-based analytics questions · Topic gap analysis · Detailed explanations

Final Review

Collector, analytics, detector recap · Dashboard design review · Monitoring pitfalls · Expert Q&A

Hands-on Labs & Projects

Collector Deployment Lab

Install and configure OpenTelemetry Collector pipelines to send infrastructure and application metrics into Splunk Observability Cloud reliably.

Metrics Analytics Workshop

Analyze real-time metrics to uncover performance issues, capacity trends, and cross-layer relationships using practical observability scenarios.

Detector Tuning Project

Create and tune detectors that balance fast incident detection with low noise across multiple environments and service types.

Dashboard Studio

Build service-focused and team-focused dashboards that visualize the right metrics clearly and remain efficient as telemetry volume grows.

Frequently Asked Questions

Is this course focused on metrics rather than logs?

Yes, the curriculum is dedicated to metrics: OpenTelemetry, analytics, detectors, and dashboards in O11y Cloud.

Do I need prior OpenTelemetry experience?

No, the course covers Collector deployment and configuration from basics to best practices.

Will I learn to build alerts and detectors that avoid noise?

Absolutely, tuning detectors and reducing false positives is a core module with hands-on projects.

Are dashboards covered for both technical and team-level visibility?

Yes, you'll design dashboards for SRE, DevOps, and engineering teams as well as executive rollups.

What roles benefit most from this certification?

Observability Engineer, SRE, DevOps Engineer, Platform Support, and Application Performance Analyst.

Can this training help teams already using Splunk O11y Cloud?

Yes, the tuning and best practices modules help optimize existing deployments.
